Background Theory

How Page Replacement Works

When physical memory is full and a new page is needed, the OS must evict a page. The algorithm choice dramatically affects performance.

FIFO↗

First In First Out

Maintains a strict queue. The oldest loaded page is always the next victim regardless of usage frequency.

O(1)Bélády's AnomalyQueue-based

LRU↗

Least Recently Used

Tracks access recency. The page unused for the longest time is evicted — exploiting temporal locality of memory access.

Optimal (Bélády's)

Requires future knowledge to evict the page needed farthest ahead. Theoretical minimum faults — the benchmark all others are measured against.
